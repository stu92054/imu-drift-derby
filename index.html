濾波參數

20 s
姿態被加速計「上」方向拉回的時間常數。τ 小 → 持續加速度被吸進重力(高通效應,軌跡走回頭路,重現三星行為);τ 大 → 淨位移保留但姿態誤差修得慢。試試 τ=2 vs τ=40 推同一段床。
OFF
兩段觸發:①儀器靜止(桌面級,室內停下拿穩即可)②鬆弛靜止(陀螺均值安靜 + GPS 確認 <0.4 m/s——穿透引擎怠速震動,GPS 確認為必要條件)。觸發後回溯扣除幻影位移 + 速度歸零 + 零偏學習。
OFF
用 OS 羅盤航向錨定 yaw 漂移(τ=20s,鎖定初始偏移後只修相對漂移)。注意:機車引擎/鋼橋/市區鋼筋會污染磁場——室內與步行實驗適用,車載實驗建議關閉對照。
ON
量測中用 watchPosition(~1Hz)記錄真實路徑當 ground truth:綠軌疊在軌跡圖(方位自動對齊 IMU)、誤差圖加綠線、CSV 含 gps_lat/lon 稀疏欄位。首次會跳定位權限請求。
OS
兩種來源同時積分:青軌=主來源(此開關選擇)、橘軌=另一來源。OFF 主用 OS acceleration(廠商高通/鉗位);ON 主用自算 includingGravity − 姿態重力(保留低頻,但姿態誤差 1° ≈ 0.17 m/s² 假加速度)。

部署到 GitHub Pages

展開步驟(iOS 感測器需要 HTTPS)
  1. 把這個檔案命名為 index.html,放進一個新的 GitHub repo(例如 imu-drift-derby)。
  2. 推上去: 
    git init
git add index.html
git commit -m "IMU drift derby"
git branch -M main
git remote add origin https://github.com/<你的帳號>/imu-drift-derby.git
git push -u origin main
  3. GitHub → repo → Settings → Pages → Source 選 Deploy from a branch → Branch 選 main / /(root) → Save。
  4. 約 1 分鐘後網址會是 https://<帳號>.github.io/imu-drift-derby/,已是 HTTPS。
  5. 用手機 Safari / Chrome 開該網址。iOS 13+ 點「開始校正」時會跳權限請求(必須由此點擊觸發)。
為什麼要 HTTPS: iOS 的 DeviceMotion 權限與 requestPermission() 只在安全來源(HTTPS 或 localhost)可用;GitHub Pages 預設就是 HTTPS。
